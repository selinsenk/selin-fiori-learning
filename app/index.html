Blank white page after clicking the button? Give it 5–10 seconds on first load — SAPUI5 is downloading from https://ui5.sap.com. If it stays blank for longer, open the console (F12) and look at the file names in the error lines:
  1. Names ending in -dbg.js (Log-dbg.js, Device-dbg.js) mean UI5 debug sources are switched on. UI5 then loads ~250 unminified files instead of ~70 bundled ones, which turns a 1-second start into minutes — or a page that never finishes on a slow connection. Fix: reload once with ?sap-ui-debug=false appended to the app URL, or press Ctrl+Shift+Alt+P in the app and untick “Use Debug Sources”.
  2. Requests for i18n_en.properties / i18n_tr.properties mean the browser is running a cached older copy of manifest.json — the current one pins supportedLocales so only i18n.properties is fetched. Fix: tick “Disable cache” on the Network tab, then hard-reload with Ctrl+Shift+R.
  3. A failed request to ui5.sap.com itself means your network is blocking the CDN.

Errors mentioning Component-preload.js, /sap/bc/lrep/ or S/CUBE are normal here and do not stop the app — see the long comment in app/taxbalance/webapp/index.html for what each one means.
